# FFmpeg — лицензирование

## Суть

FFmpeg распространяется под LGPL-2.1+, но при сборке с GPL-компонентами (например, `libx264`,
`libx265`) итоговый бинарь становится GPL. Для AVAStudio это допустимо, потому что:

- Мы **используем** FFmpeg как отдельный исполняемый файл (вызов через `spawn`), а не линкуем его
  код в свой продукт. AVAStudio не распространяет бинарь FFmpeg конечному пользователю — он работает
  на наших серверах (SaaS).
- SaaS-использование GPL-инструмента на собственной инфраструктуре не обязывает открывать исходники
  нашего продукта (GPL «срабатывает» при распространении бинарей/линковке, а не при серверном использовании).

## Сборка

- **Прод (worker, ETAP 18):** статическая сборка FFmpeg ≥ 6.0 от BtbN
  (https://github.com/BtbN/FFmpeg-Builds) с libx264/libx265/libvpx/aac — для качества и скорости.
- **Локальная разработка / CI:** системный ffmpeg (PATH) либо `FFMPEG_PATH`.

## Что нельзя

- Нельзя встраивать/распространять GPL-бинарь FFmpeg внутри скачиваемого клиентского приложения,
  не открыв исходники по GPL. У нас этого нет — FFmpeg только на сервере.
- Кодеки с патентными ограничениями (H.264/H.265) лицензированы на уровне инфраструктуры/облака;
  при масштабировании учитывать патентные пулы (MPEG-LA) — отдельно на этапе масштабирования.

## Путь к бинарю

Определяется через `env.FFMPEG_PATH` / `env.FFPROBE_PATH`, иначе системные `ffmpeg`/`ffprobe`.

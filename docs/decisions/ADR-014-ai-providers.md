# ADR-014 — Провайдеро-независимая AI-генерация (image/video/audio/music)

**Статус:** Принято · **Этап:** 11 · **Дата:** 2026-05

## Контекст
AVAStudio генерирует медиа (изображения, видео, озвучка, музыка) через внешние
AI-сервисы. Рынок волатилен: цены, лимиты и доступность провайдеров меняются,
сервисы появляются/исчезают. Привязка к одному вендору — риск (vendor lock-in,
простои, рост цен). Фаза 1 — локальная, без реальных SDK/ключей.

## Решение
Вводим унифицированный слой `@avastudio/shared/ai`: интерфейсы провайдеров по
use-case (`ImageProvider`, `VideoProvider`, `AudioProvider`, `MusicProvider`),
Zod-схемы запросов и нормализованные результаты (`AiAsset`, `AiGenerationMeta`).
Реестр (`createRegistry`) хранит упорядоченные цепочки провайдеров; вызовы идут
через `runWithFallback` — первый успешный отдаёт результат, иначе переход к
следующему, при полном отказе — `AggregateAiError` с логом попыток.

Видео — асинхронной природы: `submitVideo` → `pollVideo` (queued/processing/
succeeded/failed), чтобы переживать долгие задачи через очереди (TASK 11.5).

**Минимум 2 провайдера на use-case** (план Фазы 2, fallback-приоритет сверху вниз):
- image: OpenAI `gpt-image-1` → Replicate Flux → fal.ai
- video: Runway Gen-3 → Luma → Kling
- audio (TTS): ElevenLabs → Cartesia → OpenAI TTS
- music: Suno → Udio

## Фаза 1 (сейчас)
Только mock-драйверы (`createMock*Provider`) — детерминированные, без сети/ключей.
`createFailingImageProvider` — для проверки перехода по цепочке в тестах. Реальные
драйверы реализуют те же интерфейсы и подключаются в Фазе 2 без переписывания
вызывающего кода.

## Последствия
+ Смена/добавление провайдера = новый драйвер + регистрация, бизнес-логика не меняется.
+ Деградация устойчива: упал вендор — fallback на следующий.
+ Локальная разработка/тесты не требуют ключей.
− Нормализация к общему результату теряет часть вендор-специфичных фич (приемлемо).

## Альтернативы
- Прямые вызовы SDK в коде фич — отклонено (lock-in, дубли, хрупкость).
- Сторонний AI-gateway-сервис — отклонено для Фазы 1 (внешняя зависимость, цена).

# ADR-019 — Хостинг worker'ов (Fly.io)

**Статус:** принято · **Дата:** 2026-06 · **Этап:** 18.1 (Фаза 2)
**Связано:** ЭТАП 7 (worker), ЭТАП 6 (FFmpeg), ЭТАП 12 (PhonePool/Playwright), ЭТАП 16 (Supabase регион).

> Нумерация: ADR-018 уже занят (secret-manager) → workers-host получает ADR-019.

## Контекст
Worker выполняет FFmpeg-рендер, Playwright-браузеры (постинг/варминг) и обращения к
PhonePool/прокси. Это долгоживущие CPU/RAM-интенсивные процессы — **serverless не подходит**
(нет постоянного процесса, лимиты времени/размера, нет системного ffmpeg/Chromium).
Нужен контейнерный хостинг с авто-масштабированием по глубине очереди (BullMQ).

## Решение
**Fly.io** на старте. Контейнеры (наш `apps/worker/Dockerfile`), регионы рядом с Supabase
(`fra`), auto-stop/auto-start машин, раздельные приложения для CPU- и браузерных воркеров.

## Рассмотренные альтернативы
- **Fly.io (выбрано).** Простой деплой контейнеров, авто-скейл, machines API для кастомного
  скейлера, регионы у Supabase. Минус — дороже «голого железа» на больших объёмах.
- **Hetzner.** Дёшево и мощно, но скейл требует своего Nomad/k8s — больше операционной работы.
  Кандидат на переезд при росте (cost-этап 25).
- **AWS Fargate.** Зрелость + S3 рядом, но дороже и сложнее в настройке; привязка к AWS.

## Пулы воркеров
- **cpu-pool** (FFmpeg-рендер/уникализация): высокий CPU, умеренная RAM. Concurrency ~ vCPU.
- **browser-pool** (Playwright постинг/варминг): больше RAM (Chromium), ниже concurrency.
- **gpu-pool** (позже): NVENC-рендер при росте — отдельный класс машин с GPU.
Очереди BullMQ уже разделены по типам задач (ЭТАП 5), поэтому пулы маппятся на очереди.

## Оценка стоимости (порядково)
- ~1k активных юзеров: 1–2 cpu-машины + 1 browser-машина в среднем, авто-сон в простое →
  десятки $/мес.
- ~100k юзеров: десятки машин в пике, авто-скейл по backlog; основной драйвер цены — минуты
  рендера и браузер-сессии. Точную unit-экономику считает ЭТАП 25.6.

## Последствия
- Деплой — образом из `apps/worker/Dockerfile` (TASK 18.2), конфиг — `fly.toml` (TASK 18.3).
- Секреты — из Doppler (service-token), не в образе.
- Переносимость сохранена: образ обычный OCI, переезд на Hetzner/Fargate = смена раннера, не кода.

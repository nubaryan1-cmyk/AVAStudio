# ADR-010: Запуск FFmpeg — spawn с массивом аргументов

## Status

Accepted — 2026-05-25

## Context

Нужно вызывать FFmpeg из Node для рендера/уникализации видео. Варианты: `fluent-ffmpeg`
(обёртка с fluent-API) или прямой `child_process.spawn` с массивом аргументов.

## Decision

**Прямой `spawn` с массивом аргументов.**

## Alternatives Considered

- **fluent-ffmpeg** — удобный API, но: давно почти не поддерживается, прячет реальные аргументы,
  добавляет зависимость, усложняет точечный контроль фильтров (а уникализация требует тонкого
  filter_complex).
- **exec/shell-строка** — отклонено: риск shell-инъекций через имена файлов/параметры.

## Consequences

### Positive

- Полный контроль над аргументами FFmpeg (важно для пресетов/filter_complex — ETAP 6.3–6.5).
- Нет shell → нет инъекций (аргументы передаются массивом).
- Минимум зависимостей.
- Прямой доступ к stderr для парсинга прогресса и ошибок.

### Negative

- Сами аргументы строим вручную (но это и нужно для гибкости).

### Risks

- Нужно аккуратно собирать фильтры структурно (решается билдером пресетов в TASK 6.3).
